# Техническое задание (ТЗ)
# **TravelGo — сайт путешествий / бронирования**

---

## **1. Общая информация о проекте**
**Название:** TravelGo — сервис поиска туров, путешествий и бронирования отелей.  
**Тип:** маркетинговый лендинг + каталог + карточка тура.  
**Цель:** показать умение работать с визуально насыщенными сайтами, c большими изображениями, видеоблоками, карточками, формами поиска, анимациями и оптимизацией перформанса.

**Стек:**
- HTML5 — семантика
- CSS (ITCSS + BEM)
- Vanilla JavaScript (фильтры, поиск, модалы)
- `<picture>` + WebP / AVIF
- GitHub Pages

Проект должен выглядеть как сайт туристического сервиса уровня Booking / Airbnb / Onlinetours.

---

## **2. Структура страниц**
Основная страница: `index.html`.

Дополнительно: `tour.html` (карточка тура).

### **2.1. Главная страница (index.html)**
Секции:
1. Header (логотип + навигация)
2. Hero (фото или видео)
3. Поисковая форма «Куда поедем?»
4. Популярные направления (карточки)
5. Каталог туров
6. Преимущества TravelGo
7. Блок «Почему выбирают нас»
8. Галерея путешествий (masonry/grid)
9. Отзывы
10. FAQ
11. Подписка на рассылку
12. Footer

### **2.2. Страница тура (tour.html)**
Секции:
- Hero тура (фото/галерея)
- Основная информация: страна, название, длительность, цена
- Программа тура (accordion)
- Что включено / не включено
- Фото-галерея
- Форма бронирования

---

## **3. Основной функционал**
### **3.1. Поисковая форма**
Поля:
- Куда (input / select)
- Дата начала (date)
- Дата окончания
- Количество человек

Функционал:
- Валидация
- Перенаправление к каталогу (скролл или фильтр)

### **3.2. Каталог туров**
Каждый тур содержит:
- Большое фото
- Название
- Страна
- Цена
- Количество дней
- Краткое описание

Дополнительно:
- Кнопка «Подробнее» → `tour.html`

### **3.3. Фильтры каталога (JS)**
- По стране
- По цене (range)
- По длительности
- По рейтингу

Фильтрация — без перезагрузки.

### **3.4. Галерея (JS / CSS)**
Форматы:
- Masonry (CSS columns) или
- Grid-gallery с разными размерами фото

Lazy-loading обязательно.

### **3.5. Отзывы**
- Карусель / горизонтальная прокрутка
- Аватар + имя + рейтинг
- Текст + дата

### **3.6. FAQ (аккордеон)**
- aria-атрибуты
- плавное раскрытие

### **3.7. Форма подписки**
- Email + кнопка
- Валидация
- Отправка через fetch
- aria-live

---

## **4. Страница тура (tour.html)**
### **4.1. Галерея**
- `<picture>` + WebP/AVIF
- Слайдер (можно Swiper или кастом)

### **4.2. Информация о туре**
- Название, страна
- Цена
- Продолжительность
- Уровень сложности
- Бейджи: «Популярно», «Супер-цена»

### **4.3. Программа тура**
- Accordion: день 1, день 2…

### **4.4. Что включено / не включено**
- Иконки + текст

### **4.5. Форма бронирования**
Поля:
- Имя
- Телефон
- Email
- Даты поездки
- Количество человек

Функционал:
- Валидация
- fetch → mock endpoint
- состояния: loading/success/error

---

## **5. Адаптивность**
Mobile-first.

### Breakpoints:
- <600px — mobile
- 600–1024px — tablet
- >1024px — desktop

### Адаптивные изменения:
- Каталог: 1 → 2 → 3 колонки
- Галерея: 2 → 3 → 4 столбца
- Hero: вертикально на mobile, слева текст — справа фото на desktop
- Меню: бургер на мобильных

---

## **6. UX и анимации**
Обязательные элементы:
- Scroll reveal для секций
- Hover-эффекты карточек
- Анимации кнопок (transform)
- Плавные transitions: 150–250ms
- Skeleton loading каталога (по желанию)
- prefers-reduced-motion

---

## **7. Доступность (a11y)**
- Правильная семантика
- Один `<h1>` на страницу
- Контрастные focus-состояния
- Accordion доступен с клавиатуры
- Burger-меню управляемое клавиатурой
- aria-live у форм

---

## **8. Оптимизация и перформанс**
- Использование `<picture>`
- Lazy-loading изображений
- Предзагрузка hero-изображения
- aspect-ratio для всех карточек
- Минификация CSS/JS
- Lighthouse: A11y — 100, Performance — 90+

---

## **9. Критерии готовности**
1. Полностью адаптивный лендинг.
2. Рабочая поисковая форма.
3. Анимации без лагов.
4. Каталог туров с фильтрами.
5. Галерея (masonry или grid).
6. Отзывы + FAQ.
7. Страница тура полностью рабочая.
8. Формы отправляются через fetch.
9. Все интерактивные элементы доступны с клавиатуры.
10. Оптимизация изображений и lazy-loading.
